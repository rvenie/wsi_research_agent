# Dataset Research Agent 🔬

**Dataset Research Agent** — это автономный AI-агент, предназначенный для поиска и каталогизации общедоступных наборов данных (датасетов) с гистологическими слайдами. Агент сканирует научные репозитории, порталы открытых данных и специализированные медицинские архивы, чтобы найти и предоставить исследователям прямые ссылки на скачивание данных.

Он специализируется на поиске файлов в форматах **.svs**, **.tiff**, **.ndpi** и других форматах Whole Slide Image (WSI).

---

## 🚀 Ключевые возможности

* **Автоматический поиск:** Агент регулярно сканирует известные публичные источники (такие как TCGA, Kaggle, Zenodo, Mendeley Data и др.) на предмет новых датасетов.
* **Идентификация форматов:** Распознает и фильтрует датасеты, содержащие файлы гистологических изображений (`.svs`, `.tiff` и т.д.).
* **Извлечение метаданных:** Автоматически собирает и структурирует ключевую информацию о каждом наборе данных.
* **Прямые ссылки:** Предоставляет прямые ссылки на скачивание архивов или инструкции по доступу к данным.
* **Вывод данных:** Генерирует структурированный отчет (в .csv или .json) с полной информацией.

---

## 📋 Какие данные собирает Агент

Для каждого найденного датасета агент стремится собрать следующую информацию:

| Характеристика | Описание | Пример |
| :--- | :--- | :--- |
| **Название датасета** | Официальное название набора данных. | `Breast Cancer Histology (BACH)` |
| **Источник** | Платформа, где размещен датасет. | `Kaggle` |
| **Ссылка на источник** | URL для доступа к данным. | `https://...` |
| **Форматы файлов** | Типы найденных изображений. | `.tiff`, `.svs` |
| **Объем датасета** | Общий размер набора данных. | `~1.5 ГБ` |
| **Кол-во слайдов/изображений** | Общее число гистологических слайдов. | `400 изображений` |
| **Метаданные** | Доступная сопутствующая информация. | **Орган:** `Молочная железа`<br>**Окраска:** `H&E`<br>**Диагноз:** `4 классификации` |
| **Краткое описание** | Цель сбора данных или связанная задача. | `Классификация на 4 типа: Normal, Benign, In situ, Invasive` |

## ⚙️ Как это работает

Агент использует многоступенчатый процесс:

1.  **Скрапинг (Scraping):** Посещает предопределенный список веб-ресурсов и API.
2.  **Анализ (Parsing):** Анализирует HTML-страницы и JSON-ответы API для поиска ключевых слов (np. "histology", "WSI", "svs", "dataset").
3.  **Валидация (Validation):** Переходит по ссылкам, чтобы проверить наличие файлов в нужных форматах и доступность их для скачивания.
4.  **Извлечение (Extraction):** Собирает метаданные (объем, описание, тип окраски) со страницы датасета.
5.  **Агрегация (Aggregation):** Сохраняет всю найденную информацию в единую базу данных или файл.
